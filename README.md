# 就绪度 (Readiness) POC

这是一个用于评估用户每日“就绪度 (Readiness)”的运行在 iOS 上的概念验证 (Proof of Concept) 项目。此项目通过读取本地设备上的 HealthKit 真实数据（包括步数、睡眠时长、环境光照等），实时计算并展现用户目前的身体和精神筹备情况。

## 什么是就绪度 (Readiness)？

就绪度是一个用来评估身体当前状态的综合指标。它通过分析近期的活动水平、睡眠恢复情况和外部节律（日照等）因素，帮助用户了解自己：
- 白天：是否有充足的能量去应对工作和运动？
- 夜间：是否积累了足够的睡眠压力，适合进入休息和恢复状态？

## 核心功能

- **双模式昼夜评估 (Day / Night Mode)**
  - 根据时间自动切换（06:00 - 21:59 为清醒模式，22:00 - 05:59 为睡眠模式）。
  - 不同模式下因子权重和评价标准不同，真实反映昼夜节律带来的生理变化需求。
- **三大核心因子计分模型**
  - **节律与日照 (Circadian):** 结合当前时间的日照时长及时间估算，推测环境对机体活跃度的影响。
  - **活动与代谢 (Activity):** 结合当日步数和活动卡路里，计算日常消耗。
  - **睡眠与恢复 (Recovery):** 利用过去的睡眠时间和生理指标（如静息心率等）评估身体恢复程度。
- **睡眠欠账修正 (Sleep Debt Bonus)**
  - 在夜间模式中引入修正算法：如果昨夜睡眠不足，今晚会额外增加“睡眠就绪度”的分数，因为身体有更强的睡眠渴望（困倦感更高）。
- **Debug 数据模拟面板**
  - 无需真实 HealthKit 数据，即可通过右上角面板测试各种极限数值以及文案和圆环反馈，极大方便测试。
